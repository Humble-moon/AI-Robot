# 智扫通机器人智能客服系统 - 使用说明

## 项目简介

智扫通机器人智能客服系统是基于大语言模型（LLM）的智能问答系统，结合RAG（检索增强生成）和Agent（智能体）技术，为用户提供扫地机器人相关的实时问答、故障排查、使用建议和个性化报告生成服务。

## 技术栈

- **核心语言**：Python 3.10+
- **AI框架**：LangChain（智能体构建、工具集成）
- **模型服务**：通义千问（ChatTongyi、DashScopeEmbeddings）
- **向量存储**：Chroma DB（文档向量化、相似性检索）
- **前端交互**：Streamlit（实时聊天界面、流式输出）
- **工具库**：Pandas（数据处理）、PDF/TXT解析器（文档处理）

## 快速开始

### 环境要求

- Python 3.10+
- pip 20.0+

### 安装步骤

1. **克隆项目**
   ```bash
   git clone <项目地址>
   cd PythonHeimaProject
   ```

2. **安装依赖**
   ```bash
   pip install streamlit langchain langchain_chroma langchain_community pandas pypdf
   ```

3. **配置API密钥**
   - 本项目使用通义千问模型，需要配置DashScope API密钥
   - 在环境变量中设置API密钥：
     - Windows: `set DASHSCOPE_API_KEY=your_api_key`
     - Linux/Mac: `export DASHSCOPE_API_KEY=your_api_key`

4. **启动应用**
   ```bash
   python -m streamlit run app.py
   ```

5. **访问应用**
   - 本地：http://localhost:8501
   - 网络：http://[IP地址]:8501

## 功能使用指南

### 1. 智能问答

- **基本用法**：在聊天输入框中输入您的问题，系统会自动分析并生成回答
- **示例问题**：
  - "如何清洁扫地机器人"
  - "扫地机器人迷路了怎么办"
  - "扫地机器人电池续航多久"
  - "如何选择适合的扫地机器人"

### 2. 工具功能

系统集成了多种工具，可通过自然语言命令调用：

#### 2.1 产品比较
- **功能**：比较不同型号扫地机器人的功能和价格
- **使用示例**："比较X1和X2型号的扫地机器人"

#### 2.2 价格查询
- **功能**：获取扫地机器人的价格信息
- **使用示例**："X1型号扫地机器人的价格是多少"

#### 2.3 购买链接
- **功能**：生成扫地机器人的购买链接
- **使用示例**："给我X1型号的购买链接"

#### 2.4 维护保养
- **功能**：获取扫地机器人的维护保养建议
- **使用示例**："X1型号的维护保养建议"

#### 2.5 故障排除
- **功能**：获取扫地机器人的故障排除指南
- **使用示例**："扫地机器人不充电怎么办"

#### 2.6 使用技巧
- **功能**：获取扫地机器人的使用技巧
- **使用示例**："X1型号的使用技巧"

### 3. 系统设置

在侧边栏中可以进行以下设置：

- **主题选择**：选择喜欢的界面主题
- **清除对话历史**：清除当前的对话记录
- **常见问题**：快速访问常见问题

### 4. 文档管理

系统会自动处理`data/`目录下的文档：

- **支持的文件类型**：PDF、TXT
- **文档处理**：系统会自动解析、分块、向量化并存储到向量库
- **更新文档**：只需将新文档放入`data/`目录，系统会自动处理

## 配置说明

### 1. 配置文件

项目使用YAML格式的配置文件，存放在`config/`目录下：

- **agent.yml**：智能体相关配置
- **chroma.yml**：Chroma向量存储相关配置
- **prompts.yml**：提示词相关配置
- **rag.yml**：RAG相关配置

### 2. 环境变量

可以通过环境变量覆盖配置文件中的设置：

- **RAG_CHAT_MODEL_NAME**：聊天模型名称
- **RAG_EMBEDDING_MODEL_NAME**：嵌入模型名称
- **CHROMA_COLLECTION_NAME**：向量存储集合名称
- **CHROMA_PERSIST_DIRECTORY**：向量存储持久化目录
- **CHROMA_CHUNK_SIZE**：文档分块大小
- **CHROMA_CHUNK_OVERLAP**：分块重叠大小
- **CHROMA_K**：检索文档数量
- **AGENT_EXTERNAL_DATA_PATH**：外部数据路径

## 常见问题

### 1. 系统启动失败
- **原因**：可能是API密钥未配置或配置错误
- **解决方案**：检查环境变量中的API密钥设置

### 2. 回答不准确
- **原因**：可能是文档未加载或向量库未更新
- **解决方案**：确保`data/`目录下有相关文档，系统会自动处理

### 3. 响应速度慢
- **原因**：可能是模型调用延迟或文档处理量大
- **解决方案**：耐心等待，系统正在处理您的请求

## 扩展与定制

### 1. 添加新工具

在`agent/tools/agent_tools.py`中添加新的工具函数，使用`@tool`装饰器标记。

### 2. 添加新文档

将新文档放入`data/`目录，系统会自动处理并添加到向量库。

### 3. 修改配置

根据需要修改`config/`目录下的配置文件。

### 4. 更换模型

修改`model/factory.py`中的模型初始化代码，或修改`config/rag.yml`中的模型配置。
